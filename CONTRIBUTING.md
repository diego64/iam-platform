# Contribuindo

1. Toda mudança nasce de uma SPEC em `specs/` (ver roadmap).
2. Branch com prefixo válido: `feat/003-rbac`, `fix/login-timing`, etc.
3. Commits em Conventional Commits PT-BR — o hook `commit-msg` valida.
4. PR usando o template; gates do CI precisam estar verdes.
5. Endpoints novos exigem atualização de `openapi/openapi.yaml` e da collection Postman.
